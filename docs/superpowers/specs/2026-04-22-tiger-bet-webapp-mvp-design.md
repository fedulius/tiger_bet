# Tiger Bet WebApp MVP — дизайн (v1)

Дата: 2026-04-22
Проект: tiger_bet
Статус: Approved (user)

## 1) Цель MVP
Сделать webApp-кабинет для быстрого просмотра рекомендованных матчей и истории прогнозов, где персонализация строится через избранные виды спорта и лиги.

MVP-фокус:
- минимальный путь к полезному прогнозу;
- удобная настройка избранного;
- единый адаптивный интерфейс (mobile + desktop);
- без авторизации на первом релизе.

## 2) Информационная архитектура
Формат: один длинный экран (single-page), без отдельной навигации по страницам.

Порядок блоков на главной:
1. Рекомендованные матчи
2. Избранные виды спорта
3. Избранные лиги
4. Последние прогнозы

Верхняя зона:
- sticky-хедер с якорями к 4 блокам;
- кнопка «Обновить сейчас».

## 3) Компоненты и UX-решения
### 3.1 Рекомендованные матчи
- Карточка формата B (сбалансированная):
  - матч;
  - лига;
  - время;
  - краткая мысль прогноза;
  - уверенность.
- Количество карточек на главной: 3.
- Сортировка по умолчанию: по времени матча (ближайшие сверху).
- Обновление: авто + ручное («Обновить сейчас»).
- Новые рекомендации после автообновления: бейдж «Новые» (временная пометка).
- Empty state рекомендаций: fallback с общими топ-матчами.

### 3.2 Избранное (спорт/лиги)
- На главной в блоках есть кнопка «+ Добавить».
- По клику открывается модальное окно «Управление избранным»:
  - поиск;
  - чекбоксы выбора;
  - применение изменений.
- Удаление элемента из избранного — быстрый крестик на чипе в блоке.

### 3.3 Последние прогнозы
- Карточка формата A (минимум):
  - матч;
  - лига;
  - время;
  - краткая мысль;
  - уверенность.
- Empty state истории: текст + CTA «Открыть рекомендации».

### 3.4 Детали прогноза
- Кнопка «Подробнее» у рекомендации открывает внутреннюю страницу webApp (не внешний редирект).
- На странице деталей:
  - матч/лига/время;
  - главная мысль;
  - уверенность + основание;
  - кнопка «Открыть источник» (внешний переход на stavka.tv);
  - «Назад к рекомендациям».

## 4) Визуальный стиль
- Тема: Auto (под системную light/dark тему устройства).
- Плотность интерфейса: сбалансированная.
- Приоритет платформ: одинаковый (полноценный адаптив сразу для mobile и desktop).

## 5) Технические рамки MVP
- Авторизация: отсутствует на MVP-этапе.
- Отдельный блок пользовательских фильтров рекомендаций: не включается в MVP (только автолента по избранному).

## 6) Поведение и состояния
### 6.1 Основные состояния
- loading: подгрузка рекомендаций/истории;
- success: данные загружены;
- empty: пустые блоки по правилам empty-state;
- error: сервис недоступен/таймаут.

### 6.2 Ошибки (MVP-подход)
- Ненавязчивый inline/баннер в блоке с кнопкой повторной попытки.
- Для ручного обновления: явный короткий статус (обновлено / ошибка).
- Не блокировать весь экран из-за сбоя одного блока.

## 7) Data flow (логический)
1. Пользователь открывает webApp.
2. UI запрашивает:
   - рекомендации;
   - избранные виды спорта/лиги;
   - последние прогнозы.
3. Рекомендации собираются на основе избранного (или fallback на общие топ-матчи при пустом персональном наборе).
4. Автообновление периодически перезапрашивает рекомендации; ручное обновление делает то же по кнопке.
5. Новые элементы помечаются бейджем «Новые» в течение короткого окна.

## 8) Что сознательно вне MVP
- Полноценная авторизация и мульти-пользовательская персонализация.
- Расширенные фильтры на главной.
- Усложнённая аналитика карточек (риск, расширенные метрики и т.д.).

## 9) Критерии приемки MVP (UX)
- Пользователь заходит и сразу видит 3 рекомендации.
- Может обновить рекомендации вручную и заметить новые карточки.
- Может редактировать избранное через модалку с поиском и чекбоксами.
- Может открыть детали прогноза на внутренней странице и перейти к источнику.
- При пустых данных UI ведёт себя предсказуемо (fallback/CTA согласно правилам).
