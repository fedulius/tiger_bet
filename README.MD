# tiger_bet

tiger_bet — Telegram-бот и парсер матчей со stavka.tv.

Текущая цель MVP: пользователь выбирает матч в Telegram, а бот возвращает краткий прогноз по данным страницы матча.

## Что работает сейчас

- Поток бота: category -> league -> match -> forecast.
- Для callback-кнопок используется token-flow (короткие токены через `CallbackStore`, без длинных URL в callback_data).
- При выборе матча бот получает страницу `https://stavka.tv/matches/...` и формирует ответ.
- В прогнозе приоритет отдается редакционному блоку страницы:
  - сначала ищется «Выбор редакции»,
  - затем «Основной прогноз»,
  - затем заголовочный «Прогноз на матч ...».
- Из найденного блока извлекается главная мысль и отправляется пользователю (например: `Победа Xtreme с кэфом: 1.75`).
- Вероятность оценивается из коэффициента (если коэффициент найден в редакционном тексте).
- Если редакционный блок не найден, используется fallback-логика анализа сигналов страницы.

## Формат ответа бота (MVP)

Ответ в Telegram содержит:

- Матч
- Лига
- Краткий анализ (главная мысль прогноза)
- Уверенность
- Основание (`editorial-main-forecast`, `ai-llm`, `percent-signals`, `odds-implied`, `no-signal`)
- Объяснение (4-6 строк)
- Источник (URL матча)

## Точка входа

Файл: `index.js`

Сейчас запускается:

- `scheduler.matches(pg).main();`
- `bot.run(pg);`

HTTP API (`fastify.listen`) по-прежнему выключен в текущем MVP-потоке.

## Структура проекта

- `index.js` — bootstrap приложения
- `scheduler/Matches.js` — сбор и парсинг матчей по категориям
- `bot/` — Telegram-бот, контроллеры, callback-роутинг
- `bot/controller/MatchController/index.js` — обработка выбранного матча и отправка прогноза
- `bot/lib/CallbackStore.js` — токенизация callback_data
- `lib/aiForecastProvider.js` — провайдер прогноза с приоритетом редакционного блока
- `lib/forecastAnalyzer.js` — извлечение мысли прогноза и fallback-анализ
- `DataBase/Postgres.js` — обертка SQL-запросов к PostgreSQL
- `lib/categoryEndpoint.js` — маппинг `prediction_category_id -> endpoint`
- `docs/plans/` — проектные планы и чеклисты

## Требования

- Node.js 18+
- npm
- PostgreSQL

## Установка и запуск

1) Установить зависимости:

```bash
npm install
```

2) Настроить переменные окружения (`.env`) и доступ к БД.

3) Запустить приложение:

```bash
node index.js
```

## Тесты

```bash
npm test
```

Текущий набор покрывает:

- парсинг процентных/коэффициентных сигналов,
- извлечение главной мысли из «Основного прогноза»/«Выбора редакции»,
- формат объяснения 4-6 строк,
- поведение `MatchController.forecastAction`.

## Используемые таблицы БД (ядро)

- `public.prediction_category` — категории спорта и URL-slug
- `public.url` — сохраненные ссылки матчей, флаг верификации (`url_verified`)
- `public.user`, `public.language`, `external.public_user`, `external.system` — контур пользователей и внешних систем

## Важные замечания

- Это MVP, не production-ready версия.
- Рекомендуется хранить секреты только в `.env`.
- Для надежной работы прогноза важна структура страницы на stavka.tv; при изменении верстки может потребоваться адаптация парсинга.
